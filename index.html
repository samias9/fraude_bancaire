Banking Fraud Detection System

Machine Learning model for early detection of fraudulent transactions

Python Machine Learning XGBoost Data Analytics SMOTE

Business Context & Objective

In the banking context, early fraud detection is critical to protect customers and minimize financial losses. This project proposes an automated detection system based on the analysis of over 590,000 transactions.

Improvements vs Baseline

  • +133% recall improvement (27% → 63%)
  • -88% false alerts (vs Random Forest)
  • 98% overall accuracy

Added Value

  • Reduced financial losses
  • Decreased investigation workload
  • Improved customer experience

Decision Threshold Analysis

Decision threshold optimization allows adjusting the precision-recall trade-off according to business needs:

0.3 Recall: 71% | Precision: 62%

More frauds detected, more false alerts

0.4 ✓ Recall: 63% | Precision: 74%

Optimal threshold - Best compromise

0.5 Recall: 45% | Precision: 80%

Fewer false alerts, but missed frauds
