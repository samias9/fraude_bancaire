Banking Fraud Detection System

Machine Learning model for early detection of fraudulent transactions

Python Machine Learning XGBoost Data Analytics SMOTE

Business Context & Objective

In the banking context, early fraud detection is critical to protect customers and minimize financial losses. This project proposes an automated detection system based on the analysis of over 590,000 transactions.

Primary Objective

Maximize recall (detect maximum frauds) while maintaining acceptable precision to limit false alerts

Major Challenge

Extreme class imbalance: 96.5% legitimate transactions vs 3.5% fraud

Advanced Techniques

Imbalance Management

  • SMOTE (Synthetic Minority Over-sampling)
  • Oversampling ratio: 0.5
  • Class weighting (scale_pos_weight)
  • Stratified train/test split

Model Optimization

  • Decision threshold adjusted: 0.4 (vs 0.5)
  • Correlation-based feature selection
  • Dimensionality reduction (100 features)
  • Stratified cross-validation
